About the project
The B.S.N. is a software prototype that relies on a network of wifi sensors to monitor the human body vital signs. It should provide autonomous and real time response.

This is a project initiated on an academic research at Unb(University of Brasília) by Genaína Rodrigues and her students.

This server is focused on the remote configuration and execution of the BSN.
How does it work
The BSN is written on C++, for this language is able to provide a fast and reliable execution. Also the OpenDaVinci framework for real time systems was used to develop this artefact.

The system uses configuration to explicit what it expects as a healthy data. To generate and simulate data, a Markov Chain is used which is a model where the probabilitie of each state relies on the previous probabilities. You can read more about it on wikipedia here or watching this great video clicking here.

After generating, the data will be filtered using a moving average technique to remove noise. Later on it will be evaluated to calculate its risk. To understand deeper how it works please check our repositorie.

To remotely execute, the BSN relies on an API written on Python3, which provides every feature available for this server. You can check our repositories used for the BSN on our own Gits, located on the page bottom.
